## 📖 项目简介

本项目基于若依（RuoYi-Vue）前后端分离系统，完成了**功能测试**和**接口自动化测试**两大方向的测试工作。测试覆盖登录、用户管理、角色管理、部门管理四大核心模块，共设计并执行 **165个测试用例**，发现 **26个缺陷**。

## 🔍 核心缺陷摘录

### 严重缺陷（P0）

| 编号 | 缺陷描述 |
|------|----------|
| BUG-I-02 | 新增用户-用户名为空时仍能创建成功 |
| BUG-I-03 | 新增用户-昵称为空时仍能创建成功 |
| BUG-I-04 | 新增用户-密码为空时仍能创建成功 |
| BUG-I-15 | 修改用户-缺少userId时仍返回成功 |
| BUG-I-17 | 重置密码-密码为空时仍能重置成功 |
| BUG-I-20~23 | 分配角色-参数校验完全缺失 |

### 主要问题分析

1. **校验大面积缺失**：新增用户、修改用户、重置密码、分配角色等接口的必填参数校验基本缺失，导致脏数据可能污染数据库。
2. **权限联动失效**：角色禁用后，该角色下的用户仍能正常登录，权限控制存在漏洞。
3. **HTTP状态码不规范**：未携带token时返回200（应为401），不符合RESTful规范。

## 🚀 快速开始

### 1. 环境准备

- 部署若依系统
- 确保后端服务运行在 `http://192.168.163.130:8080`

### 2. 导入Postman脚本

1. 打开Postman
2. 点击 Import -> 选择 `postman/RuoYi-Interface-Testing.postman_collection.json`
3. 导入环境变量 `RuoYi-Interface-Testing.postman_environment.json`
4. 设置环境变量 `baseUrl` 为实际后端地址

### 3. 运行测试

- 先执行登录接口获取token
- 依次运行各模块批量测试脚本
- 查看Test Results获取测试结果
